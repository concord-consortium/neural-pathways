# explorer_data_with_words.json

## Generate

```
cd browser-data && conda run -n nnmaker python generate_explorer_data_with_words.py [--decimals N]
```

Output: `dist/explorer_data_with_words.json`

The `--decimals` flag controls numeric precision (default: 6). Using `--decimals 2` reduces file size from ~14 MB to ~10 MB.

## Description

Extends the explorer_data format with per-word SHAP scores showing each word's effect on each pathway. Limited to the 1,842 test reviews that have pre-computed SHAP data, with 7 pathways.

The original train CSV (3,000 reviews) is split into train (2,700) and dev (300), with separate activation files for each. The pre-computed SHAP data (`yelp_test_shap_pathway_associations_test_fa_simple.csv`) was generated by Ritam's code (`distilbert_trained_library.py`), which fits its own FA on the test-split activations at import time. To match, this generation script also fits its own StandardScaler and Factor Analysis from scratch on the test-split activations (`yelp_test_NN_activations_simple.json`, 3,000 reviews) — it does not load a saved FA model. This differs from explorer_data and viz_data, which fit on the train split.

The script uses the same random seed as Ritam's code (`random_state=0`, the sklearn default) so the FA fit is identical. scikit-learn's Factor Analysis uses randomized SVD by default for speed, so the seed matters — a different seed produces slightly different results even on the same data. With matching seeds, `pathway_scores` and `unmasked_values` agree (after rounding).

Each review includes everything from explorer_data, plus:
- **base_values**: array of 7 values, one per pathway. The pathway score when all tokens are masked (SHAP baseline).
- **unmasked_values**: array of 7 values. The pathway score with all tokens present. These match `pathway_scores` since both come from the same FA fit.
- **words**: array of word objects, each with:
  - **word**: the display text (DistilBERT subword tokens like `bis` + `##tro` are joined back into `bistro`, with SHAP scores summed)
  - **scores**: array of 7 signed SHAP values, one per pathway. Positive means the word pushes the pathway score up; negative means down.

To compute normalized impact: `impact = score / (unmasked_value - base_value)` for the corresponding pathway.
