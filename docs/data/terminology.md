# Terminology

## FA terminology mapping

Factor Analysis has standard terminology that maps to project-specific terms. The [factor-analysis.md](factor-analysis.md) document uses FA terminology throughout; this table helps translate.

| FA term | Project term | Notes |
|---------|-------------|-------|
| **factor** | **pathway** | A latent pattern discovered by FA across neuron activations |
| **observation** | **review** | Specifically, a review's 780-dimensional neuron activation vector |
| **variable** | **neuron activation** | One of the 780 activation values per review |
| **factor loadings** | **loadings** | No project-specific rename — "loadings" is used directly. A factor's loadings are the pattern of weights across all 780 neurons that define that pathway. |
| **factor scores** | **pathway scores** | Per-review values saying how strongly each pathway is expressed in that review |
| **noise variance / uniquenesses** | **noise variance** | No project-specific rename. Per-variable variance not explained by any factor. |

## Activations

The numeric outputs of neurons (or layers) in the neural network when processing a review. In this project, activations are collected from three layers of the trained DistilBERT classifier:

- **CLS embedding** (768 values): the model's internal representation of the full review
- **Classifier hidden layer 1** (6 values): first layer of the classification head
- **Classifier hidden layer 2** (6 values): second layer of the classification head

These are concatenated into a single 780-dimensional vector per review. The activation files (e.g. `yelp_test_NN_activations_simple.json`) store these pre-computed vectors so that downstream analysis doesn't need to re-run the model.

"Activations" is standard terminology in neural network research — it refers to the output values produced by neurons after applying their activation function.

## FA fit

The process of running Factor Analysis on a set of activations to discover pathways. See [factor-analysis.md](factor-analysis.md) for details on fitting, projecting, and reconstruction.

An FA fit is specific to the data it was trained on. This project has multiple FA fits depending on which split's activations are used:

- **Train-split FA fit**: used by `explorer_data.json` and `viz_data.json` (fit on 2,700 train-split activations)
- **Test-split FA fit**: used by `explorer_data_with_words.json` and the pre-computed SHAP data (fit on 3,000 test-split activations)
- **Dev-split FA fit**: used in some of Ritam's analysis files (fit on 300 dev-split activations)

The FA is fit from scratch by each generation script — there is no saved FA model file that gets loaded.

"Fitting" is general machine learning terminology — it means training or estimating a model's parameters from data.

## SHAP

SHAP (SHapley Additive exPlanations) is a method for measuring how much each input feature contributes to a model's output. It comes from game theory — Shapley values assign each player a fair share of a group's total payout. In machine learning, the "players" are input features and the "payout" is the model's prediction.

In this project, SHAP is used to measure each word's effect on each pathway score. The approach works by **masking**: for each token, SHAP replaces it with a special mask token and observes how the pathway score changes. The SHAP library (`shap.Explainer` with `shap.maskers.Text`) handles the combinatorial logic of attributing effects fairly across all tokens, rather than just doing single-token ablation.

Key values per word per pathway:
- **score**: the SHAP value — a signed number representing that word's contribution to the pathway score. Positive means the word pushes the score up; negative means down.
- **base_value**: the pathway score when all tokens are masked (the SHAP baseline — roughly what the model outputs with no information).
- **unmasked_value**: the pathway score with all tokens present.

The SHAP values for all words in a review sum to approximately `unmasked_value - base_value` for each pathway. This additive property is a guarantee of Shapley values.

The pre-computed SHAP data is in `NNmodels/distilbert_trained_correlations/yelp_test_shap_pathway_associations_*_fa_simple.csv`. These were generated by Ritam's code (`distilbert_trained_library.py`) which runs the DistilBERT model many times per review to compute the masking effects. This is computationally expensive, which is why only 1,842 of the 3,000 test reviews have SHAP data.

SHAP is a widely-used open source library, not specific to this project.

## StandardScaler

A preprocessing step applied to the 780 neuron activations before Factor Analysis. Normalizes each neuron independently to have mean=0 and standard deviation=1 across reviews. See [factor-analysis.md](factor-analysis.md) for the equations and details on why this matters.

In this project, the scaler is always fit on the same data as the FA (e.g. train-split activations for `explorer_data.json`, test-split for `explorer_data_with_words.json`). The scaler stores two 780-element vectors (mean and scale), which are included in `viz_data.json` for client-side conversion.

StandardScaler is a standard scikit-learn utility, not specific to this project.

## Train / dev / test splits

See [review-datasets.md](review-datasets.md) for descriptions of the review sets (train, dev, test, synthetic GPT) and their annotations.

In filenames, `_test_` and `_dev_` can each mean two different things:
- The reviews being analyzed are from that split (e.g. `yelp_test_NN_activations_simple.json`)
- The FA was fit on that split (e.g. `yelp_test_shap_pathway_associations_test_fa_simple.csv` — the `_test_fa_` part means "using an FA fit on test-split activations")
